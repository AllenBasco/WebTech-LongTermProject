HTTP Fundamentals
HTTP runs on top of TCP/IP, using TCP port 80 by default or TCP port 443 for HTTPS(HTTP over SSL/TLS).
HTTP is based on a client-server architecture.
clients a.k.a. user agents(UA):
web browsers, web crawlers/spiders, other end user tools and applications
HTTP provides an encryption panel
servers
origin servers
proxy servers
allows yout to authenticate pass to one another until it reaches origin server.
gateways

HTTP uses a request-response standard protocol
the client sends an HTTP request message to the server
the server processes the request and replies with an HTTP response message.
HTTP is a stateless communications protocol
servers do not keep information about clients in-between requests

8 Standard Methods
  1. GET
    2. - Transfer a current selected representation of the resource identified by the request URI
    - Fetch resources
  2. HEAD
    3. - same as GET with one difference (returns only the statusline and the headers)
    - used to retrieve metadata about the entity implied by the request without transferring the entity itself
  3. PUT
    4. - store the enclosed entity in the message body under the specified request URI
  4. OPTIONS
    5. - tell you what you can do or request information about communication
  5. TRACE
    6. - used to request a loop-back of the request message
    - used for testing/diagnostics of the request/response chain
  6. CONNECT
    7. - use only for HTTPS communication
    - request the establishment of a tunel to the destination origin server, and if successful thereafter restrict its behavior to blind forwarding of packets
Different Types of Methods
  1. Safe Method
    2. A method that doesn't make any change.
  2. Idempotent Method
    3. A method that you can use a method multiple times but get the same result.
  3. cacheable method
    4. The response generated could be cachd or not.

HTTP Status Code
Informational(1xx)
100 -continue : A status code that would allow the performing of operations.
101 -switching protocols : A status code that could switch to a higher protocol.

Success (2xx)
200 -Ok : The operation carried out
201 -created : The response to PUT request.
202 -Accepted : The request has been recieved but not yet processed.
203 -Non-Authorative : Same as above but does not come from origin or authority.
204 -No content : It is successful but does not send anything back.
205 -Reset Content : Te request has heen done and can be reset.
206 -Partial Content: It does not send a full resource but has a range.

Redirection (3xx) : Redirected from another location.
300 -Multiple Choices : Not much information was sent, so choices were given.
301 -Moved Permanently : Means that what you are looking for is now in a different resource.
302 -Found : Means that a resource was found after making a request.
303 -See other : Means to see other resources avaiilable.
304 -Not Modified : When you are doing a conditional request.
305 -Use proxy : You are not supposed to access directly from the resource but from the proxy itself.
306 -(unused) : An unused status code as of now.
307 -Temporary Redirect : It is redirected temporarily but can come back later.

Error (4xx)
400 -bad request : Malformed resource.
401 -unauthorized : Accessing from an unprotected resource.
402 -payment required : Systems like e-commerce application are needed. e.g. PayPal
403 -Forbidden : Not allowed to go to this resource.
404 -Not found : The resource is not found.
405 -Method Not allowed : Means it is not configured to allow.
406 -Not acceptable : Appropriate representation is found on the server.
407 -proxy Authentication Required : Authorization through proxy.
408 -Request Time-out : The request is taking too long to process.
409 -Conflict : You can look a resource.
410 -Gone : The resource is gone and will never come back.
411 -Length Required : It is expecting a content length but did not send.
412 -Precondition Failed : Creating a resource that does not exist, but if it did exist then it failed.
413 -Request Entity Too Large : The file is too large.
414 -Request-URI Too large : If the URI is too long.
415 -Unsupported Media Type : Sending something and specifying content type but server application cannot request.
416 -Requested Range Not Satisfied : Range is not enough.
417 -Expectation Failed : What you requested is not there at all.
426 -Upgrade Required : It requires a higher version of a protocol.

Server Error(5xx)
500 -Internal Server Error : It is misconfiguration on the server.
501 -Not Implemented : It was not implemented.
502 -Bad Gateway : There is something wrong witn the gateway.
503 -Service unavailable : The service is unavailable.
504 -Gateway Time-out : Gateway has timed out.
505 -HTTP version Not Supported : When using a version of the protocol, in which they don't know how to communicate with.
