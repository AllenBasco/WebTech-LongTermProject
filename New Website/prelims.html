THE INTERNET AND THE WORLD WIDE WEB

What is Internet?
Internet is a system of interconnected computer networks, a global medium wherein computers are connected together via standardized communication protocols such as TCP/IP. Any user can get information, if permission is granted, from one device on the fly.
Logically linked together by a globally unique address space based on the Internet Protocol(IP)
Able to support communications using the transmission control protocol/Internet Protocol
Provides uses or makes accessible either publicly or privately.
INTER
to cross boundaries
NETWORK
means connecting devices
What is Internet of Things (IoT)?
It is any appliance that can connect to the network. Can be accessed remotely. Examples of these are Smart cars, Smart Toothbrush, etc.

HTTP (HyperText Transfer Protocol)
Application layer communications protocol used to access resources(hypertext/hypermedia) on the World Wide Web.
It was invented by Tim Berners-Lee.
Jointly developed by the W3C and the IETF (Internet Engineering Task Force).
It runs on TCP/IP protocol using port 80 by default, the secure version being TCP 443 (HTTPS, integrated TLS/SSL)
What is HTTP based on?
HTTP is based on client-server architecture a.k.a clients, user-agents (web browsers, web crawlers, other end user tools and applications)
Origin Server
it is web server that contained the very First Web Page.
Proxy Server
allows you to authenticate pass to one another until it reaches origin server
Gateways
it can convert protocols and can actually perform in any network later. HTTP traffic can be directed to gateways and then sent to proxies.
Tunnels
a way in which various protocols from the network are encapsulated through the protocol of HTTP. HTTP tunnel applications are available for application software.
HTTP uses a request-response standard protocol
The client sends an HTTP request message to the server
The server processes the request and replies with an HTTP response message.
HTTP is a stateless communications protocol
Servers do not keep information about clients in-between requests
HTTP provides support for other functionalities, such as:
  • Cache Control
    • storage that is available locally (Local Storage)
  • Content Media Type
    • is a two-part identifier for file formats and format contents transmitted on the Internet.
  • Content/transfer coding
    • when you sent something it will be recieve as is
  • Content negotiation
    • if the other one can't recieve the zip file then don`t send it as a zip file. It must specify which version should fit its capabilities the best.
  • Persistent Connection
    • the idea of this is to send and recieve multiple http request using only single TCP connection
  • Request pipelining
    • one request at a time
  • Authentication/Authorization
    • configure all the server
HTTP Resource Addressing
HTTP resources are identified using URIs (RFC 3986),or, more specifically, HTTP URLs
User information or authentication credentials(deprecated)

Domain Name

Port Number
these numbers define what port you are using. The standard port of HTTP is 80.
Path to Resource
resolved to the document root on the server
May refer to a static or dynamic resource.
Query
a string is part of the Uniform Resource Locator(URL). It contains data that is not inline with the path structure.
Typically provided as key value pairs, with ampersan (&) separators between key/value pairs.
