HYPERTEXT MARKUP LANGUAGE

History of HyperText Markup Language

Version Year Description
HTML 1.0 1989 Very first HTML created by Tim-Berners Lee
HTML 2.0 1995
  • Published with an actual standard
  • RFC HTML 2.0
  • W3C was already founded (World Wide Web Consortium)
HTML 3.0 / 3.2 / 4.0 / 4.01 1999 Three Versions of HTML 4.01
  1. Strict
    • Deprecated elements/attribute
  2. Transitional
    • Can use the old elements
  3. Frameset
    • Divide the page into frames
XHTML (Extensible HyperText Markup Language) 2000
  • Reformulation of HTML 4 in XML 1.0
  • Case sensitive
  • No optional, all elements are required
  • Stricter set of authoring requirements
  • Attribute value must always be quoted
2005
  • Ian Hickson
    • Founded WHAWG

What is HTML?
HTML is a language used to create web pages - (electronic document)
Roles in Modern Web Development
  1. Structure and Content
    • Example is : <h1> Hello World </h1>
  2. Aesthetics / Presentational
    • Example is : <body bgcolor = "red">
  3. Behavioral Aspect

                        <!DOCTYPE html>
                        <html lang="en">

                        <meta charset = “UTF-8”>
                        </head>
                        <body>
                        </body>
                        </html>

Content Categories of HTML

Name of Category Description
The lang Attribute This attribute specifies the language to be used for a certain element.
The title Attribute The title attribute helps describe information of a certain element.
The href Attribute This attribute specifies the link and from where it originated.
The size Attribute This attribute specifies the number of attributes in a certain element such as characters.
The alt Attribute This attribute specifies the alternative text and would be applied if the text element is not working or compatible.

HTML Attributes

Name of Attribute Description
Metadata Content is content that sets up the presentation or behavior of the rest of the content, or that sets up the relationship of the document with other documents, or that conveys other "out of band" information. <base> , <meta> , <link> , <script>, <style>, <title>
Flow Content Most elements that are used in the body of documents and applications are categorized as flow content. <a> , <aside> , <div> , <main>, <header>, <footer> , <ul>, <pre>, <p>
Sectioning Content Content that defines the scope of headings and footers <article> , <aside> , <nav> , <section>
Heading Content Defines the header of a section (whether explicitly marked up using sectioning content elements, or implied by the heading content itself). <h1> , <h2> , <h3> , <h4>, <h5>
Phrasing Content the text of the document, as well as elements that mark up that text at the intra-paragraph level. Runs of phrasing content form paragraphs. <abbr> , <b> , <br> , <button>, <i>, <em>
Embedded Content It imports another resource or inserts content content from another markup language or namespace into the document. <audio> , <canvas> , <embed> , <iframe>, <img>, <math> , <object>, <svg>, <video>

CASCADING STYLESHEETS

What is Cascading Stylesheet?
CSS is a Language used to specify the presentation aspects (e.g., formatting, layout) of structured marked up documents
Developed by Hakon Wium Lie (CHSS) and Bert Bos (SSP)
Versions
  • CSS 1 - December 1996
  • CSS 2.1 - June 2011
  • CSS 3
CSS Processors
A program that takes one type of data and converts it to another type of data
Examples
  • SASS (Syntactically Awesome Style Sheets)
  • HALM
CSS Frameworks
A pre-prepared software framework that is meant to allow for easier, more standards-compliant web design using the Cascading Style Sheets language
Examples
  • Less Framework
  • 960 Grid System
  • Bootstrap
  • Foundation
  • Materialize
Sources of Styles
  1. Author Style
    • Style that the author of the website created
  2. User Style
    • Allow the user to override the author style
  3. User Agent Style
    • A style that is already available in the browser
AT-RULES DESCRIPTION
@charset It interprets the character set that can be applied with the style sheet.
@import It informs the CSS to add a stylesheet that is external.
@namespace Informs the CSS engine that the content must have an XML container that could hold identifiers
@media A group rule which could pertain with media as long as it meets specified conditions and content can be applied accordingly. It is also conditional.
@supports Also conditional, this group rule affects content if the browser meets specified conditions.
@document Group rule that applies content once the document stylesheets meets conditions.
@page This describes layout changes especially once he document is printed.
@font-face Describes the feature of an external font that would be downloaded.
@keyframes Shows the aspect of intermediary steps of the animation sequence in CSS.
@viewport Outlines the viewport for devices which utilize small screens.

Consists of a selector followed by a declaration block that are delimited by braces. Inside the declaration block are the property declarations.-Property name and Property Value

SELECTOR TYPES OF SELECTOR KIND/S OF SELCTOR CODE CODE DESCRIPTION
SIMPLE SELECTORS Type Selector h1, pdiv
Universal Selector *
Attribute Selector [attr=value] This is a representation of an element which has the att attribute where the value is appointed with “val”.
[attr~=value] This is a representation of an element which has the att attribute where the value is appointed with separated list of words which have whitespace.
[attr|=value] This is a representation of an element which has the att attribute where the value is begins either with “val” or is followed by an “-”.
[attr^=value] This is a representation of an element which has the att attribute where the prefix begins with “val”.
[attr$=value] This is a representation of an element which has the att attribute where the value ends with “val” as the suffix.
[attr*=value] This is a representation of an element which has the att attribute where one instance of substring “val” is contained.

Structural Pseudo-Classes :root In the document, this represents the root as an element.
:first-child this represents the first child of another element as an element.
:last-child This represents the last child of another element as an element.
:only-child This represents the parent element as an element and no other children are in the parent element.
:nth-child() This element in the document tree would have an+b-1 siblings before it.
:nth-last-child() This element in the document tree would have an+b-1 siblings after it.
:first-of-type This element would be represented with in the list of children of its own parent element that it is the type of its siblings.
:last-of-type This element would be represented with in the list of children of its own parent element that it is the type of its siblings.
:only-of-type This represents the parent element as an element and no other children are in the parent element and the element name is the same and is also expanded.
:nth-of-type() This element in the document tree would have an+b-1 siblings before it and the element is the same expanded.
:nth-last-of-type() This element in the document tree would have an+b-1 siblings after it and the element is the same expanded.
:empty In the document tree, this element has no children.
