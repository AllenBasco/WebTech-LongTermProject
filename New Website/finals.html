JAVASERVER PAGES

What Is a JSP Page?
A JSP page is a text document that has two types: Static Data and JSP elements. Static data can be expressed through any text-format like HTML, SVG, XML etc. while JSP elements build up on content that is dynamic.
JSP Construct Types
  • A content type is set with this particular directive and will brought back to the page: (<%@page ...%>)
  • A directive where custom tag libraries are imported: (<%@taglib ...%>)
  • Objects created with a package containing locales and identifiers that directs to a object as a standard element is called a jsp:bean.
  • There are object properties that call the value of JSP expressions under the language expressions and these are (${ }). Values are recycled are used to establish custom tag attribute values and build content that is dynamic.
  • A variable is determined as a custom tag it is called (c:set), a collection is emphasized locale names called (c:forEach),and tentatively HTML text is inseted in the response.
  • Another standard element that appoints a value of a certain property of an object this is the jsp:setProperty
  • This function test equality of the attributes and the collection’s present item there is also an inherited operator, ==, and is used for testing the equality. (f:equals)

Here is a JSP page example:

                          <%@ page contentType="text/html; charset=UTF-8" %>
                          <%@ taglib uri="http:/java.net/http/jsp"
                            prefix="j" %>
                          <%@ taglib uri="/functions" prefix="functions" %>
                          <html>
                          <head><title>Dates Decentralized</title></head>
                          <body bgcolor="yellow">
                          <jsp:useBean id="locales" scope="application"
                           class="mypkg.MyLocales"/>
                          <form name="localeForm" action="index.jsp" method="post">
                          <j:set var="selectedLocaleString" value="${param.locale}" />
                          <j:set var="selectedFlag"
                            value="${!empty selectedLocaleString}" />
                          <b>Locale:</b>
                          <select name=locale>
                          <j:forEach var="localeString" items="${locales.localeNames}" >
                          <j:choose>
                           <j:when test="${selectedFlag}">
                            <j:choose>
                             <j:when
                              test="${f:equals(selectedLocaleString, localeString)}" >
                              <option selected>${localeString}</option>
                             </j:when>
                             <j:otherwise>
                              <option>${localeString}</option>
                             </j:otherwise>
                            </j:choose>
                           </j:when>
                           <j:otherwise>
                            <option>${localeString}</option>
                           </j:otherwise>
                          </j:choose>
                          </j:forEach>
                          </select>
                          <input type="submit" name="Submit" value="Get Date">
                          </form>
                          <j:if test="${selectedFlag}" >
                           <jsp:setProperty name="locales"
                            property="selectedLocaleString"
                            value="${selectedLocaleString}" />
                           <jsp:useBean id="date" class="mypkg.MyDate"/>
                           <jsp:setProperty name="date" property="locale"
                            value="${locales.selectedLocale}"/>
                           <b>Date: </b>${date.date}</c:if>
                          </body>
                          </html>

JSTL SQL Actions
The JavaServer Pages Standard Tag Library (JSTL) is a collection of useful JSP tags which encapsulates the core functionality common to many JSP applications.

These are the following:

JSTL SQL ACTIONS Description
<sql:dateParam> Used as a nested action for the <sql:query>
<sql:update> To supply a date and time value for a value placeholder.
<sql:transactions> Provides nested database action elements with a shared Connection, set up to execute all statements as one transaction.
<sql:query> Executes the SQL query defined in its body or through the sql attribute.
<sql:update> Executes the SQL update defined in its body or through the sql attribute.
<sql:param> Sets a parameter in an SQL statement to the specified value.
<sql:setDataSource> Creates a simple DataSource suitable only for prototyping

    OWASP Top 10 Most Critical Web Application Security Risk
    A robust awareness document used for security in the web application. It amounts to the comprehensive consensus about crucial security risks to web applications. This list was shared and compiled by experts all over the world to make this list and identify crucial problems.
    1. Injection
      • Untrusted Data sent to interpreter as part of a query.
    2. Broken Authentication and Session Management
      • can be implemented in a manner that is not correct. This allows hackers a chance to compromise passwords, keys, and manymore.
    3. Cross-Site Scripting (XSS)
      • untrusted data is included in a webpage without proper validation. Hackers can utilize XSS to hijack the victim’s browser.
    4. Broken Access Control
      • restrictions what users are permitted to do. Attackers can exploit the functionality that is unauthorized.
    5. Security Misconfiguration
      • Good security needs a properly defined framework/s. Setting should be secure and up to date.
    6. Sensitive Data Exposure
      • Many web applications do not protect data that is sensitive.
    7. Insufficient Attack Protection
      • Many applications lack the ability to prevent, detect, and respond to attacks that are manual and automated.
    8. Cross-Site Request Forgery (CSRF)
    9. Using Components with Known Vulnerabilities
      • Encodes the current session data as a session encoded string
    10. Underprotected APIs
      • Perform session data garbage collection
